Get UNC running on your machine and compile your first model in minutes.

## Prerequisites

Before you begin, ensure you have:

- **macOS** with Apple Silicon (M1/M2/M3/M4) recommended
- **Xcode Command Line Tools** for Metal compiler
- **Rust toolchain** (1.70 or later)

<Note>
UNC currently supports macOS with Metal backend. CUDA, ROCm, and WASM backends are planned.
</Note>

## Installation

<Steps>
  <Step title="Install Xcode Command Line Tools">
    ```bash
    xcode-select --install
    ```
    
    Verify installation:
    ```bash
    clang --version
    # Apple clang version 15.0.0 (clang-1500.0.40.1)
    ```
  </Step>

  <Step title="Install Rust">
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```
    
    Follow the prompts, then reload your shell and verify:
    ```bash
    rustc --version
    # rustc 1.75.0 (82e1608df 2023-12-21)
    ```
  </Step>

  <Step title="Clone and build UNC">
    ```bash
    git clone https://github.com/pacifio/unc.git
    cd unc
    cargo build --release
    ```
    
    The binary will be at `target/release/unc`. Add it to your PATH:
    ```bash
    export PATH="$PWD/target/release:$PATH"
    ```
  </Step>

  <Step title="Verify installation">
    ```bash
    unc list-architectures
    ```
    
    You should see:
    ```
    Supported architectures:
      LlamaForCausalLM               LLaMA, LLaMA-2, LLaMA-3
      MistralForCausalLM             Mistral, Mixtral
      Qwen2ForCausalLM               Qwen, Qwen-2
      PhiForCausalLM                 Phi-2, Phi-3
      GemmaForCausalLM               Gemma, Gemma-2
    ```
  </Step>
</Steps>

## Your First Compilation

Let's compile TinyLlama 1.1B with Q4_0 quantization:

```bash
unc compile \
  --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 \
  --quant q4-0 \
  -o ./tinyllama
```

<Note>
The first run will download the model from HuggingFace (~2.2 GB). Subsequent compilations will use the cached weights.
</Note>

You'll see output like:

```
unc: compiling TinyLlama/TinyLlama-1.1B-Chat-v1.0 → ./tinyllama.unc
  arch=LLaMA, layers=22, heads=32 (kv=4), hidden=2048, ffn=5632
  graph: 483 nodes, quant: q4_0
  registry: 47 kernels for Metal(M1Pro)
  compile: 341 nodes, 38 kernels, activation=128 MB, kv=256 MB
Done → ./tinyllama.unc
```

## Run Inference

Now run inference on your compiled model:

```bash
unc run ./tinyllama.unc --prompt "The history of" --max-tokens 200
```

Output:

```
The history of the United States is a rich tapestry woven from diverse threads 
of exploration, colonization, revolution, expansion, and social transformation. 
From the indigenous peoples who first inhabited the land to the modern era of 
globalization and technological innovation, the American story is one of constant 
evolution and reinvention...

200 tokens in 1.32s (152.0 tok/s)
```

<Tip>
Q4_0 quantization delivers the best throughput: **152 tok/s** on M1 Pro with only **0.6 GB** of weights.
</Tip>

## What Just Happened?

When you ran `unc compile`, UNC:

1. **Downloaded the model** from HuggingFace (config.json + safetensors)
2. **Parsed the config** to detect LLaMA architecture and hyperparameters
3. **Built an IR graph** with 483 typed tensor operations
4. **Applied optimization passes**: QKV fusion, SwiGLU fusion, memory planning
5. **Quantized weights** from BF16 to Q4_0 (4 bits per weight)
6. **Matched Metal kernels** for your M1 Pro GPU
7. **Generated a .unc bundle** with Metal shaders, weights, and metadata

When you ran `unc run`, UNC:

1. **Loaded the bundle** and compiled Metal shaders via clang (cached after first run)
2. **Initialized the Metal device** and allocated GPU buffers
3. **Loaded quantized weights** (618 MB Q4_0 → GPU memory)
4. **Tokenized the prompt** using the embedded tokenizer
5. **Ran prefill** (parallel attention over prompt tokens)
6. **Ran decode** (autoregressive sampling, one token per step)

## Output Modes

UNC supports two compilation modes:

<CardGroup cols={2}>
  <Card title="JIT Bundle (.unc)" icon="file-zipper">
    **Development mode** - Fast iteration, JIT-compiled via clang
    
    - Separate weights + metallib
    - ~544 KB binary + HF weights
    - Requires clang at runtime
    - First run: ~2s, cached: ~1.7s
  </Card>
  
  <Card title="AOT Binary (Mach-O)" icon="file-binary">
    **Production mode** - Zero dependencies, embedded weights
    
    - Standalone executable
    - ~2.1 GB self-contained
    - No runtime dependencies
    - Startup: ~1.5s
    
    ```bash
    unc compile --model TinyLlama/... --quant q4-0 --binary -o ./tinyllama
    ./tinyllama --prompt "Hello" --max-tokens 50
    ```
  </Card>
</CardGroup>

## Quantization Options

Choose the right tradeoff for your use case:

| Quantization | Size (1.1B) | Speed (M1 Pro) | Use Case |
|--------------|-------------|----------------|----------|
| `f16` | 2.2 GB | 47.9 tok/s | Maximum quality |
| `q8-0` | 1.1 GB | 76.6 tok/s | Balanced |
| `q4-0` | 0.6 GB | **152.0 tok/s** | Maximum speed |

<Tip>
Start with `q4-0` for the best throughput. Quality loss is typically &lt;1% perplexity increase.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command not found: unc">
    Add UNC to your PATH:
    ```bash
    export PATH="/path/to/unc/target/release:$PATH"
    ```
    
    Or install globally:
    ```bash
    cargo install --path .
    ```
  </Accordion>

  <Accordion title="clang not found">
    Install Xcode Command Line Tools:
    ```bash
    xcode-select --install
    ```
  </Accordion>

  <Accordion title="Out of memory">
    Reduce context length or use a smaller model:
    ```bash
    unc compile --model TinyLlama/... --quant q4-0 --max-seq-len 2048
    ```
  </Accordion>

  <Accordion title="Slow inference">
    Check thermal throttling and GPU utilization. Close background apps and ensure you're using Q4_0 quantization for best speed.
  </Accordion>
</AccordionGroup>
